83    /**
84     * Method that does a batch call on Deletes, Gets and Puts. The ordering of
85     * execution of the actions is not defined. Meaning if you do a Put and a

87     * guaranteed that the Get returns what the Put had put.
88     *

94     * @since 0.90.0
95     */

300    * <p>

310    */

318    * readers do not take row locks so get and scan operations can see this
319    * operation partially completed.
320    *
321    * @param increment object that specifies the columns and amounts to be used

333    * <b>true</b>)}
334    * @param row The row that contains the cell to increment.
335    * @param family The column family of the cell to increment.
336    * @param qualifier The column qualifier of the cell to increment.
337    * @param amount The amount to increment the cell with (or decrement, if the
338    * amount is negative).
339    * @return The new value, post increment.
340    * @throws IOException if a remote or network exception occurs.

345   /**
346    * Atomically increments a column value. If the column value already exists
347    * and is not a big-endian long, this could throw an exception. If the column
348    * value does not yet exist it is initialized to <code>amount</code> and
349    * written to the specified column.
350    *
351    * <p>Setting writeToWAL to false means that in a fail scenario, you will lose
352    * any increments that have not been flushed.
353    * @param row The row that contains the cell to increment.
354    * @param family The column family of the cell to increment.
355    * @param qualifier The column qualifier of the cell to increment.
356    * @param amount The amount to increment the cell with (or decrement, if the
357    * amount is negative).
358    * @param writeToWAL if {@code true}, the operation will be applied to the
359    * Write Ahead Log (WAL).  This makes the operation slower but safer, as if
360    * the call returns successfully, it is guaranteed that the increment will
361    * be safely persisted.  When set to {@code false}, the call may return
362    * successfully before the increment is safely persisted, so it's possible
363    * that the increment be lost in the event of a failure happening before the
364    * operation gets persisted.
365    * @return The new value, post increment.
366    * @throws IOException if a remote or network exception occurs.
367    */
368   long incrementColumnValue(byte[] row, byte[] family, byte[] qualifier,
369       long amount, boolean writeToWAL) throws IOException;
