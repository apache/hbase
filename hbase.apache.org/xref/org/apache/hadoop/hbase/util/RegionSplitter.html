68  /**
69   * The {@link RegionSplitter} class provides several utilities to help in the
70   * administration lifecycle for developers who choose to manually split regions
71   * instead of having HBase handle that automatically. The most useful utilities
72   * are:
73   * <p>
74   * <ul>
75   * <li>Create a table with a specified number of pre-split regions
76   * <li>Execute a rolling split of all regions on an existing table
77   * </ul>
78   * <p>
79   * Both operations can be safely done on a live server.
80   * <p>
81   * <b>Question:</b> How do I turn off automatic splitting? <br>
82   * <b>Answer:</b> Automatic splitting is determined by the configuration value
83   * <i>HConstants.HREGION_MAX_FILESIZE</i>. It is not recommended that you set this
84   * to Long.MAX_VALUE in case you forget about manual splits. A suggested setting
85   * is 100GB, which would result in > 1hr major compactions if reached.
86   * <p>
87   * <b>Question:</b> Why did the original authors decide to manually split? <br>
88   * <b>Answer:</b> Specific workload characteristics of our use case allowed us
89   * to benefit from a manual split system.
90   * <p>
91   * <ul>
92   * <li>Data (~1k) that would grow instead of being replaced
93   * <li>Data growth was roughly uniform across all regions
94   * <li>OLTP workload. Data loss is a big deal.
95   * </ul>
96   * <p>
97   * <b>Question:</b> Why is manual splitting good for this workload? <br>
98   * <b>Answer:</b> Although automated splitting is not a bad option, there are
99   * benefits to manual splitting.
100  * <p>
101  * <ul>
102  * <li>With growing amounts of data, splits will continually be needed. Since
103  * you always know exactly what regions you have, long-term debugging and
104  * profiling is much easier with manual splits. It is hard to trace the logs to
105  * understand region level problems if it keeps splitting and getting renamed.
106  * <li>Data offlining bugs + unknown number of split regions == oh crap! If an
107  * HLog or StoreFile was mistakenly unprocessed by HBase due to a weird bug and
108  * you notice it a day or so later, you can be assured that the regions
109  * specified in these files are the same as the current regions and you have
110  * less headaches trying to restore/replay your data.
111  * <li>You can finely tune your compaction algorithm. With roughly uniform data
112  * growth, it's easy to cause split / compaction storms as the regions all
113  * roughly hit the same data size at the same time. With manual splits, you can
114  * let staggered, time-based major compactions spread out your network IO load.
115  * </ul>
116  * <p>
117  * <b>Question:</b> What's the optimal number of pre-split regions to create? <br>
118  * <b>Answer:</b> Mileage will vary depending upon your application.
119  * <p>
120  * The short answer for our application is that we started with 10 pre-split
121  * regions / server and watched our data growth over time. It's better to err on
122  * the side of too little regions and rolling split later.
123  * <p>
124  * The more complicated answer is that this depends upon the largest storefile
125  * in your region. With a growing data size, this will get larger over time. You
126  * want the largest region to be just big enough that the {@link HStore} compact
127  * selection algorithm only compacts it due to a timed major. If you don't, your
128  * cluster can be prone to compaction storms as the algorithm decides to run
129  * major compactions on a large series of regions all at once. Note that
130  * compaction storms are due to the uniform data growth, not the manual split
131  * decision.
132  * <p>
133  * If you pre-split your regions too thin, you can increase the major compaction
134  * interval by configuring HConstants.MAJOR_COMPACTION_PERIOD. If your data size
135  * grows too large, use this script to perform a network IO safe rolling split
136  * of all regions.
137  */
138 @InterfaceAudience.Private
139 public class RegionSplitter {
140   static final Log LOG = LogFactory.getLog(RegionSplitter.class);

142   /**
143    * A generic interface for the RegionSplitter code to use for all it's
144    * functionality. Note that the original authors of this code use
145    * {@link HexStringSplit} to partition their table and set it as default, but
146    * provided this for your custom algorithm. To use, create a new derived class
147    * from this interface and call {@link RegionSplitter#createPresplitTable} or
148    * {@link RegionSplitter#rollingSplit(String, SplitAlgorithm, Configuration)} with the
149    * argument splitClassName giving the name of your class.
150    */
151   public static interface SplitAlgorithm {
152     /**
153      * Split a pre-existing region into 2 regions.
154      *
155      * @param start
156      *          first row (inclusive)
157      * @param end
158      *          last row (exclusive)
159      * @return the split row to use
160      */
161     byte[] split(byte[] start, byte[] end);
162 
163     /**
164      * Split an entire table.
165      *
166      * @param numRegions
167      *          number of regions to split the table into
168      *
169      * @throws RuntimeException
170      *           user input is validated at this time. may throw a runtime
171      *           exception in response to a parse failure
172      * @return array of split keys for the initial regions of the table. The
173      *         length of the returned array should be numRegions-1.
174      */
175     byte[][] split(int numRegions);
176 
177     /**
178      * In HBase, the first row is represented by an empty byte array. This might
179      * cause problems with your split algorithm or row printing. All your APIs
180      * will be passed firstRow() instead of empty array.
181      *
182      * @return your representation of your first row
183      */
184     byte[] firstRow();
185 
186     /**
187      * In HBase, the last row is represented by an empty byte array. This might
188      * cause problems with your split algorithm or row printing. All your APIs
189      * will be passed firstRow() instead of empty array.
190      *
191      * @return your representation of your last row
192      */
193     byte[] lastRow();
194 
195     /**
196      * In HBase, the last row is represented by an empty byte array. Set this
197      * value to help the split code understand how to evenly divide the first
198      * region.
199      *
200      * @param userInput
201      *          raw user input (may throw RuntimeException on parse failure)
202      */
203     void setFirstRow(String userInput);
204 
205     /**
206      * In HBase, the last row is represented by an empty byte array. Set this
207      * value to help the split code understand how to evenly divide the last
208      * region. Note that this last row is inclusive for all rows sharing the
209      * same prefix.
210      *
211      * @param userInput
212      *          raw user input (may throw RuntimeException on parse failure)
213      */
214     void setLastRow(String userInput);
215 
216     /**
217      * @param input
218      *          user or file input for row
219      * @return byte array representation of this row for HBase
220      */
221     byte[] strToRow(String input);
222 
223     /**
224      * @param row
225      *          byte array representing a row in HBase
226      * @return String to use for debug & file printing
227      */
228     String rowToStr(byte[] row);
229 
230     /**
231      * @return the separator character to use when storing / printing the row
232      */
233     String separator();
234   }

236   /**
237    * The main function for the RegionSplitter application. Common uses:
238    * <p>
239    * <ul>
240    * <li>create a table named 'myTable' with 60 pre-split regions containing 2
241    * column families 'test' & 'rs', assuming the keys are hex-encoded ASCII:
242    * <ul>
243    * <li>bin/hbase org.apache.hadoop.hbase.util.RegionSplitter -c 60 -f test:rs
244    * myTable HexStringSplit
245    * </ul>
246    * <li>perform a rolling split of 'myTable' (i.e. 60 => 120 regions), # 2
247    * outstanding splits at a time, assuming keys are uniformly distributed
248    * bytes:
249    * <ul>
250    * <li>bin/hbase org.apache.hadoop.hbase.util.RegionSplitter -r -o 2 myTable
251    * UniformSplit
252    * </ul>
253    * </ul>
254    *
255    * There are two SplitAlgorithms built into RegionSplitter, HexStringSplit
256    * and UniformSplit. These are different strategies for choosing region
257    * boundaries. See their source code for details.
258    *
259    * @param args
260    *          Usage: RegionSplitter &lt;TABLE&gt; &lt;SPLITALGORITHM&gt;
261    *          &lt;-c &lt;# regions&gt; -f &lt;family:family:...&gt; | -r
262    *          [-o &lt;# outstanding splits&gt;]&gt;
263    *          [-D &lt;conf.param=value&gt;]
264    * @throws IOException
265    *           HBase IO problem
266    * @throws InterruptedException
267    *           user requested exit
268    * @throws ParseException
269    *           problem parsing user input
270    */
271   @SuppressWarnings("static-access")
272   public static void main(String[] args) throws IOException,
273       InterruptedException, ParseException {
274     Configuration conf = HBaseConfiguration.create();
275 
276     // parse user input
277     Options opt = new Options();
278     opt.addOption(OptionBuilder.withArgName("property=value").hasArg()
279         .withDescription("Override HBase Configuration Settings").create("D"));
280     opt.addOption(OptionBuilder.withArgName("region count").hasArg()
281         .withDescription(
282             "Create a new table with a pre-split number of regions")
283         .create("c"));
284     opt.addOption(OptionBuilder.withArgName("family:family:...").hasArg()
285         .withDescription(
286             "Column Families to create with new table.  Required with -c")
287         .create("f"));
288     opt.addOption("h", false, "Print this usage help");
289     opt.addOption("r", false, "Perform a rolling split of an existing region");
290     opt.addOption(OptionBuilder.withArgName("count").hasArg().withDescription(
291         "Max outstanding splits that have unfinished major compactions")
292         .create("o"));
293     opt.addOption(null, "firstrow", true,
294         "First Row in Table for Split Algorithm");
295     opt.addOption(null, "lastrow", true,
296         "Last Row in Table for Split Algorithm");
297     opt.addOption(null, "risky", false,
298         "Skip verification steps to complete quickly."
299             + "STRONGLY DISCOURAGED for production systems.  ");
300     CommandLine cmd = new GnuParser().parse(opt, args);
301 
302     if (cmd.hasOption("D")) {
303       for (String confOpt : cmd.getOptionValues("D")) {
304         String[] kv = confOpt.split("=", 2);
305         if (kv.length == 2) {
306           conf.set(kv[0], kv[1]);
307           LOG.debug("-D configuration override: " + kv[0] + "=" + kv[1]);
308         } else {
309           throw new ParseException("-D option format invalid: " + confOpt);
310         }
311       }
312     }
313 
314     if (cmd.hasOption("risky")) {
315       conf.setBoolean("split.verify", false);
316     }
317 
318     boolean createTable = cmd.hasOption("c") && cmd.hasOption("f");
319     boolean rollingSplit = cmd.hasOption("r");
320     boolean oneOperOnly = createTable ^ rollingSplit;
321 
322     if (2 != cmd.getArgList().size() || !oneOperOnly || cmd.hasOption("h")) {
323       new HelpFormatter().printHelp("RegionSplitter <TABLE> <SPLITALGORITHM>\n"+
324 		  "SPLITALGORITHM is a java class name of a class implementing " +
325 		  "SplitAlgorithm, or one of the special strings HexStringSplit " +
326 		  "or UniformSplit, which are built-in split algorithms. " +
327 		  "HexStringSplit treats keys as hexadecimal ASCII, and " +
328 		  "UniformSplit treats keys as arbitrary bytes.", opt);
329       return;
330     }
331     String tableName = cmd.getArgs()[0];
332     String splitClass = cmd.getArgs()[1];
333     SplitAlgorithm splitAlgo = newSplitAlgoInstance(conf, splitClass);
334 
335     if (cmd.hasOption("firstrow")) {
336       splitAlgo.setFirstRow(cmd.getOptionValue("firstrow"));
337     }
338     if (cmd.hasOption("lastrow")) {
339       splitAlgo.setLastRow(cmd.getOptionValue("lastrow"));
340     }
341 
342     if (createTable) {
343       conf.set("split.count", cmd.getOptionValue("c"));
344       createPresplitTable(tableName, splitAlgo, cmd.getOptionValue("f").split(":"), conf);
345     }
346 
347     if (rollingSplit) {
348       if (cmd.hasOption("o")) {
349         conf.set("split.outstanding", cmd.getOptionValue("o"));
350       }
351       rollingSplit(tableName, splitAlgo, conf);
352     }
353   }

355   static void createPresplitTable(String tableName, SplitAlgorithm splitAlgo,
356           String[] columnFamilies, Configuration conf) throws IOException,
357           InterruptedException {
358     final int splitCount = conf.getInt("split.count", 0);
359     Preconditions.checkArgument(splitCount > 1, "Split count must be > 1");
360 
361     Preconditions.checkArgument(columnFamilies.length > 0,
362         "Must specify at least one column family. ");
363     LOG.debug("Creating table " + tableName + " with " + columnFamilies.length
364         + " column families.  Presplitting to " + splitCount + " regions");
365 
366     HTableDescriptor desc = new HTableDescriptor(tableName);
367     for (String cf : columnFamilies) {
368       desc.addFamily(new HColumnDescriptor(Bytes.toBytes(cf)));
369     }
370     HBaseAdmin admin = new HBaseAdmin(conf);
371     Preconditions.checkArgument(!admin.tableExists(tableName),
372         "Table already exists: " + tableName);
373     admin.createTable(desc, splitAlgo.split(splitCount));
374     admin.close();
375     LOG.debug("Table created!  Waiting for regions to show online in META...");
376     if (!conf.getBoolean("split.verify", true)) {
377       // NOTE: createTable is synchronous on the table, but not on the regions
378       int onlineRegions = 0;
379       while (onlineRegions < splitCount) {
380         onlineRegions = MetaReader.getRegionCount(conf, tableName);
381         LOG.debug(onlineRegions + " of " + splitCount + " regions online...");
382         if (onlineRegions < splitCount) {
383           Thread.sleep(10 * 1000); // sleep
384         }
385       }
386     }
387 
388     LOG.debug("Finished creating table with " + splitCount + " regions");
389   }

391   static void rollingSplit(String tableName, SplitAlgorithm splitAlgo,
392           Configuration conf) throws IOException, InterruptedException {
393     final int minOS = conf.getInt("split.outstanding", 2);
394 
395     HTable table = new HTable(conf, tableName);
396 
397     // max outstanding splits. default == 50% of servers
398     final int MAX_OUTSTANDING =
399         Math.max(table.getConnection().getCurrentNrHRS() / 2, minOS);
400 
401     Path hbDir = new Path(conf.get(HConstants.HBASE_DIR));
402     Path tableDir = HTableDescriptor.getTableDir(hbDir, table.getTableName());
403     Path splitFile = new Path(tableDir, "_balancedSplit");
404     FileSystem fs = FileSystem.get(conf);
405 
406     // get a list of daughter regions to create
407     LinkedList<Pair<byte[], byte[]>> tmpRegionSet = getSplits(table, splitAlgo);
408     LinkedList<Pair<byte[], byte[]>> outstanding = Lists.newLinkedList();
409     int splitCount = 0;
410     final int origCount = tmpRegionSet.size();
411 
412     // all splits must compact & we have 1 compact thread, so 2 split
413     // requests to the same RS can stall the outstanding split queue.
414     // To fix, group the regions into an RS pool and round-robin through it
415     LOG.debug("Bucketing regions by regionserver...");
416     TreeMap<String, LinkedList<Pair<byte[], byte[]>>> daughterRegions =
417       Maps.newTreeMap();
418     for (Pair<byte[], byte[]> dr : tmpRegionSet) {
419       String rsLocation = table.getRegionLocation(dr.getSecond()).
420         getHostnamePort();
421       if (!daughterRegions.containsKey(rsLocation)) {
422         LinkedList<Pair<byte[], byte[]>> entry = Lists.newLinkedList();
423         daughterRegions.put(rsLocation, entry);
424       }
425       daughterRegions.get(rsLocation).add(dr);
426     }
427     LOG.debug("Done with bucketing.  Split time!");
428     long startTime = System.currentTimeMillis();
429 
430     // open the split file and modify it as splits finish
431     FSDataInputStream tmpIn = fs.open(splitFile);
432     byte[] rawData = new byte[tmpIn.available()];
433     tmpIn.readFully(rawData);
434     tmpIn.close();
435     FSDataOutputStream splitOut = fs.create(splitFile);
436     splitOut.write(rawData);
437 
438     try {
439       // *** split code ***
440       while (!daughterRegions.isEmpty()) {
441         LOG.debug(daughterRegions.size() + " RS have regions to splt.");
442 
443         // Get RegionServer : region count mapping
444         final TreeMap<ServerName, Integer> rsSizes = Maps.newTreeMap();
445         Map<HRegionInfo, ServerName> regionsInfo = table.getRegionLocations();
446         for (ServerName rs : regionsInfo.values()) {
447           if (rsSizes.containsKey(rs)) {
448             rsSizes.put(rs, rsSizes.get(rs) + 1);
449           } else {
450             rsSizes.put(rs, 1);
451           }
452         }
453 
454         // sort the RS by the number of regions they have
455         List<String> serversLeft = Lists.newArrayList(daughterRegions .keySet());
456         Collections.sort(serversLeft, new Comparator<String>() {
457           public int compare(String o1, String o2) {
458             return rsSizes.get(o1).compareTo(rsSizes.get(o2));
459           }
460         });
461 
462         // round-robin through the RS list. Choose the lightest-loaded servers
463         // first to keep the master from load-balancing regions as we split.
464         for (String rsLoc : serversLeft) {
465           Pair<byte[], byte[]> dr = null;
466 
467           // find a region in the RS list that hasn't been moved
468           LOG.debug("Finding a region on " + rsLoc);
469           LinkedList<Pair<byte[], byte[]>> regionList = daughterRegions
470               .get(rsLoc);
471           while (!regionList.isEmpty()) {
472             dr = regionList.pop();
473 
474             // get current region info
475             byte[] split = dr.getSecond();
476             HRegionLocation regionLoc = table.getRegionLocation(split);
477 
478             // if this region moved locations
479             String newRs = regionLoc.getHostnamePort();
480             if (newRs.compareTo(rsLoc) != 0) {
481               LOG.debug("Region with " + splitAlgo.rowToStr(split)
482                   + " moved to " + newRs + ". Relocating...");
483               // relocate it, don't use it right now
484               if (!daughterRegions.containsKey(newRs)) {
485                 LinkedList<Pair<byte[], byte[]>> entry = Lists.newLinkedList();
486                 daughterRegions.put(newRs, entry);
487               }
488               daughterRegions.get(newRs).add(dr);
489               dr = null;
490               continue;
491             }
492 
493             // make sure this region wasn't already split
494             byte[] sk = regionLoc.getRegionInfo().getStartKey();
495             if (sk.length != 0) {
496               if (Bytes.equals(split, sk)) {
497                 LOG.debug("Region already split on "
498                     + splitAlgo.rowToStr(split) + ".  Skipping this region...");
499                 ++splitCount;
500                 dr = null;
501                 continue;
502               }
503               byte[] start = dr.getFirst();
504               Preconditions.checkArgument(Bytes.equals(start, sk), splitAlgo
505                   .rowToStr(start) + " != " + splitAlgo.rowToStr(sk));
506             }
507 
508             // passed all checks! found a good region
509             break;
510           }
511           if (regionList.isEmpty()) {
512             daughterRegions.remove(rsLoc);
513           }
514           if (dr == null)
515             continue;
516 
517           // we have a good region, time to split!
518           byte[] split = dr.getSecond();
519           LOG.debug("Splitting at " + splitAlgo.rowToStr(split));
520           HBaseAdmin admin = new HBaseAdmin(table.getConfiguration());
521           admin.split(table.getTableName(), split);
522 
523           LinkedList<Pair<byte[], byte[]>> finished = Lists.newLinkedList();
524           if (conf.getBoolean("split.verify", true)) {
525             // we need to verify and rate-limit our splits
526             outstanding.addLast(dr);
527             // with too many outstanding splits, wait for some to finish
528             while (outstanding.size() >= MAX_OUTSTANDING) {
529               finished = splitScan(outstanding, table, splitAlgo);
530               if (finished.isEmpty()) {
531                 Thread.sleep(30 * 1000);
532               } else {
533                 outstanding.removeAll(finished);
534               }
535             }
536           } else {
537             finished.add(dr);
538           }
539 
540           // mark each finished region as successfully split.
541           for (Pair<byte[], byte[]> region : finished) {
542             splitOut.writeChars("- " + splitAlgo.rowToStr(region.getFirst())
543                 + " " + splitAlgo.rowToStr(region.getSecond()) + "\n");
544             splitCount++;
545             if (splitCount % 10 == 0) {
546               long tDiff = (System.currentTimeMillis() - startTime)
547                   / splitCount;
548               LOG.debug("STATUS UPDATE: " + splitCount + " / " + origCount
549                   + ". Avg Time / Split = "
550                   + org.apache.hadoop.util.StringUtils.formatTime(tDiff));
551             }
552           }
553         }
554       }
555       if (conf.getBoolean("split.verify", true)) {
556         while (!outstanding.isEmpty()) {
557           LinkedList<Pair<byte[], byte[]>> finished = splitScan(outstanding,
558               table, splitAlgo);
559           if (finished.isEmpty()) {
560             Thread.sleep(30 * 1000);
561           } else {
562             outstanding.removeAll(finished);
563             for (Pair<byte[], byte[]> region : finished) {
564               splitOut.writeChars("- " + splitAlgo.rowToStr(region.getFirst())
565                   + " " + splitAlgo.rowToStr(region.getSecond()) + "\n");
566             }
567           }
568         }
569       }
570       LOG.debug("All regions have been successfully split!");
571     } finally {
572       long tDiff = System.currentTimeMillis() - startTime;
573       LOG.debug("TOTAL TIME = "
574           + org.apache.hadoop.util.StringUtils.formatTime(tDiff));
575       LOG.debug("Splits = " + splitCount);
576       LOG.debug("Avg Time / Split = "
577           + org.apache.hadoop.util.StringUtils.formatTime(tDiff / splitCount));
578 
579       splitOut.close();
580       if (table != null){
581         table.close();
582       }
583     }
584     fs.delete(splitFile, false);
585   }

587   /**
588    * @throws IOException if the specified SplitAlgorithm class couldn't be
589    * instantiated
590    */
591   public static SplitAlgorithm newSplitAlgoInstance(Configuration conf,
592           String splitClassName) throws IOException {
593     Class<?> splitClass;
594 
595     // For split algorithms builtin to RegionSplitter, the user can specify
596     // their simple class name instead of a fully qualified class name.
597     if(splitClassName.equals(HexStringSplit.class.getSimpleName())) {
598       splitClass = HexStringSplit.class;
599     } else if (splitClassName.equals(UniformSplit.class.getSimpleName())) {
600       splitClass = UniformSplit.class;
601     } else {
602       try {
603         splitClass = conf.getClassByName(splitClassName);
604       } catch (ClassNotFoundException e) {
605         throw new IOException("Couldn't load split class " + splitClassName, e);
606       }
607       if(splitClass == null) {
608         throw new IOException("Failed loading split class " + splitClassName);
609       }
610       if(!SplitAlgorithm.class.isAssignableFrom(splitClass)) {
611         throw new IOException(
612                 "Specified split class doesn't implement SplitAlgorithm");
613       }
614     }
615     try {
616       return splitClass.asSubclass(SplitAlgorithm.class).newInstance();
617     } catch (Exception e) {
618       throw new IOException("Problem loading split algorithm: ", e);
619     }

622   static LinkedList<Pair<byte[], byte[]>> splitScan(
623       LinkedList<Pair<byte[], byte[]>> regionList, HTable table,
624       SplitAlgorithm splitAlgo)
625       throws IOException, InterruptedException {
626     LinkedList<Pair<byte[], byte[]>> finished = Lists.newLinkedList();
627     LinkedList<Pair<byte[], byte[]>> logicalSplitting = Lists.newLinkedList();
628     LinkedList<Pair<byte[], byte[]>> physicalSplitting = Lists.newLinkedList();
629 
630     // get table info
631     Path hbDir = new Path(table.getConfiguration().get(HConstants.HBASE_DIR));
632     Path tableDir = HTableDescriptor.getTableDir(hbDir, table.getTableName());
633     FileSystem fs = tableDir.getFileSystem(table.getConfiguration());
634 
635     // clear the cache to forcibly refresh region information
636     table.clearRegionCache();
637 
638     // for every region that hasn't been verified as a finished split
639     for (Pair<byte[], byte[]> region : regionList) {
640       byte[] start = region.getFirst();
641       byte[] split = region.getSecond();
642 
643       // see if the new split daughter region has come online
644       try {
645         HRegionInfo dri = table.getRegionLocation(split).getRegionInfo();
646         if (dri.isOffline() || !Bytes.equals(dri.getStartKey(), split)) {
647           logicalSplitting.add(region);
648           continue;
649         }
650       } catch (NoServerForRegionException nsfre) {
651         // NSFRE will occur if the old META entry has no server assigned
652         LOG.info(nsfre);
653         logicalSplitting.add(region);
654         continue;
655       }
656 
657       try {
658         // when a daughter region is opened, a compaction is triggered
659         // wait until compaction completes for both daughter regions
660         LinkedList<HRegionInfo> check = Lists.newLinkedList();
661         check.add(table.getRegionLocation(start).getRegionInfo());
662         check.add(table.getRegionLocation(split).getRegionInfo());
663         for (HRegionInfo hri : check.toArray(new HRegionInfo[] {})) {
664           boolean refFound = false;
665           byte[] sk = hri.getStartKey();
666           if (sk.length == 0)
667             sk = splitAlgo.firstRow();
668           String startKey = splitAlgo.rowToStr(sk);
669           HTableDescriptor htd = table.getTableDescriptor();
670           // check every Column Family for that region
671           for (HColumnDescriptor c : htd.getFamilies()) {
672             Path cfDir = HStore.getStoreHomedir(tableDir, hri.getEncodedName(),
673                 c.getName());
674             if (fs.exists(cfDir)) {
675               for (FileStatus file : fs.listStatus(cfDir)) {
676                 refFound |= StoreFile.isReference(file.getPath());
677                 if (refFound)
678                   break;
679               }
680             }
681             if (refFound)
682               break;
683           }
684           // compaction is completed when all reference files are gone
685           if (!refFound) {
686             check.remove(hri);
687           }
688         }
689         if (check.isEmpty()) {
690           finished.add(region);
691         } else {
692           physicalSplitting.add(region);
693         }
694       } catch (NoServerForRegionException nsfre) {
695         LOG.debug("No Server Exception thrown for: "
696             + splitAlgo.rowToStr(start));
697         physicalSplitting.add(region);
698         table.clearRegionCache();
699       }
700     }
701 
702     LOG.debug("Split Scan: " + finished.size() + " finished / "
703         + logicalSplitting.size() + " split wait / "
704         + physicalSplitting.size() + " reference wait");
705 
706     return finished;
707   }

709   static LinkedList<Pair<byte[], byte[]>> getSplits(HTable table,
710       SplitAlgorithm splitAlgo) throws IOException {
711     Path hbDir = new Path(table.getConfiguration().get(HConstants.HBASE_DIR));
712     Path tableDir = HTableDescriptor.getTableDir(hbDir, table.getTableName());
713     Path splitFile = new Path(tableDir, "_balancedSplit");
714     FileSystem fs = tableDir.getFileSystem(table.getConfiguration());
715 
716     // using strings because (new byte[]{0}).equals(new byte[]{0}) == false
717     Set<Pair<String, String>> daughterRegions = Sets.newHashSet();
718 
719     // does a split file exist?
720     if (!fs.exists(splitFile)) {
721       // NO = fresh start. calculate splits to make
722       LOG.debug("No _balancedSplit file.  Calculating splits...");
723 
724       // query meta for all regions in the table
725       Set<Pair<byte[], byte[]>> rows = Sets.newHashSet();
726       Pair<byte[][], byte[][]> tmp = table.getStartEndKeys();
727       Preconditions.checkArgument(
728           tmp.getFirst().length == tmp.getSecond().length,
729           "Start and End rows should be equivalent");
730       for (int i = 0; i < tmp.getFirst().length; ++i) {
731         byte[] start = tmp.getFirst()[i], end = tmp.getSecond()[i];
732         if (start.length == 0)
733           start = splitAlgo.firstRow();
734         if (end.length == 0)
735           end = splitAlgo.lastRow();
736         rows.add(Pair.newPair(start, end));
737       }
738       LOG.debug("Table " + Bytes.toString(table.getTableName()) + " has "
739           + rows.size() + " regions that will be split.");
740 
741       // prepare the split file
742       Path tmpFile = new Path(tableDir, "_balancedSplit_prepare");
743       FSDataOutputStream tmpOut = fs.create(tmpFile);
744 
745       // calculate all the splits == [daughterRegions] = [(start, splitPoint)]
746       for (Pair<byte[], byte[]> r : rows) {
747         byte[] splitPoint = splitAlgo.split(r.getFirst(), r.getSecond());
748         String startStr = splitAlgo.rowToStr(r.getFirst());
749         String splitStr = splitAlgo.rowToStr(splitPoint);
750         daughterRegions.add(Pair.newPair(startStr, splitStr));
751         LOG.debug("Will Split [" + startStr + " , "
752             + splitAlgo.rowToStr(r.getSecond()) + ") at " + splitStr);
753         tmpOut.writeChars("+ " + startStr + splitAlgo.separator() + splitStr
754             + "\n");
755       }
756       tmpOut.close();
757       fs.rename(tmpFile, splitFile);
758     } else {
759       LOG.debug("_balancedSplit file found. Replay log to restore state...");
760       FSUtils.getInstance(fs, table.getConfiguration())
761         .recoverFileLease(fs, splitFile, table.getConfiguration());
762 
763       // parse split file and process remaining splits
764       FSDataInputStream tmpIn = fs.open(splitFile);
765       StringBuilder sb = new StringBuilder(tmpIn.available());
766       while (tmpIn.available() > 0) {
767         sb.append(tmpIn.readChar());
768       }
769       tmpIn.close();
770       for (String line : sb.toString().split("\n")) {
771         String[] cmd = line.split(splitAlgo.separator());
772         Preconditions.checkArgument(3 == cmd.length);
773         byte[] start = splitAlgo.strToRow(cmd[1]);
774         String startStr = splitAlgo.rowToStr(start);
775         byte[] splitPoint = splitAlgo.strToRow(cmd[2]);
776         String splitStr = splitAlgo.rowToStr(splitPoint);
777         Pair<String, String> r = Pair.newPair(startStr, splitStr);
778         if (cmd[0].equals("+")) {
779           LOG.debug("Adding: " + r);
780           daughterRegions.add(r);
781         } else {
782           LOG.debug("Removing: " + r);
783           Preconditions.checkArgument(cmd[0].equals("-"),
784               "Unknown option: " + cmd[0]);
785           Preconditions.checkState(daughterRegions.contains(r),
786               "Missing row: " + r);
787           daughterRegions.remove(r);
788         }
789       }
790       LOG.debug("Done reading. " + daughterRegions.size() + " regions left.");
791     }
792     LinkedList<Pair<byte[], byte[]>> ret = Lists.newLinkedList();
793     for (Pair<String, String> r : daughterRegions) {
794       ret.add(Pair.newPair(splitAlgo.strToRow(r.getFirst()), splitAlgo
795           .strToRow(r.getSecond())));
796     }
797     return ret;
798   }

800   /**
801    * HexStringSplit is a well-known {@link SplitAlgorithm} for choosing region
802    * boundaries. The format of a HexStringSplit region boundary is the ASCII
803    * representation of an MD5 checksum, or any other uniformly distributed
804    * hexadecimal value. Row are hex-encoded long values in the range
805    * <b>"00000000" => "FFFFFFFF"</b> and are left-padded with zeros to keep the
806    * same order lexicographically as if they were binary.
807    *
808    * Since this split algorithm uses hex strings as keys, it is easy to read &
809    * write in the shell but takes up more space and may be non-intuitive.
810    */
811   public static class HexStringSplit implements SplitAlgorithm {
812     final static String DEFAULT_MIN_HEX = "00000000";
813     final static String DEFAULT_MAX_HEX = "FFFFFFFF";
814 
815     String firstRow = DEFAULT_MIN_HEX;
816     BigInteger firstRowInt = BigInteger.ZERO;
817     String lastRow = DEFAULT_MAX_HEX;
818     BigInteger lastRowInt = new BigInteger(lastRow, 16);
819     int rowComparisonLength = lastRow.length();
820 
821     public byte[] split(byte[] start, byte[] end) {
822       BigInteger s = convertToBigInteger(start);
823       BigInteger e = convertToBigInteger(end);
824       Preconditions.checkArgument(!e.equals(BigInteger.ZERO));
825       return convertToByte(split2(s, e));
826     }
827 
828     public byte[][] split(int n) {
829       Preconditions.checkArgument(lastRowInt.compareTo(firstRowInt) > 0,
830           "last row (%s) is configured less than first row (%s)", lastRow,
831           firstRow);
832       // +1 to range because the last row is inclusive
833       BigInteger range = lastRowInt.subtract(firstRowInt).add(BigInteger.ONE);
834       Preconditions.checkState(range.compareTo(BigInteger.valueOf(n)) >= 0,
835           "split granularity (%s) is greater than the range (%s)", n, range);
836 
837       BigInteger[] splits = new BigInteger[n - 1];
838       BigInteger sizeOfEachSplit = range.divide(BigInteger.valueOf(n));
839       for (int i = 1; i < n; i++) {
840         // NOTE: this means the last region gets all the slop.
841         // This is not a big deal if we're assuming n << MAXHEX
842         splits[i - 1] = firstRowInt.add(sizeOfEachSplit.multiply(BigInteger
843             .valueOf(i)));
844       }
845       return convertToBytes(splits);
846     }
